SmartPager

Your AI-Powered Planning Assistant

SmartPager is a physical pager that listens, understands, and schedules. Audio captured via Raspberry Pi hardware is transcribed with Whisper, interpreted by GPT-4 into events, tasks, and study blocks, and then placed by an optimization-driven scheduler around fixed calendar items. It writes back to Google Calendar so the e-ink screen always shows your next item and today's agenda.

Motivation

Motivation

Students burn time juggling calendars, Courseworks, and to-do lists; small slips turn into conflicts. We’re chasing planning that feels effortless and dependable: hands-full voice capture on the pager, distinct handling of fixed events vs tasks vs study blocks, auto-resolves when users move things, and matching agendas on the e-ink screen and Google Calendar. It’s meant to be reliable every day, keeping up with shifting deadlines without extra overhead.

System

SmartPager blends embedded hardware, cloud intelligence, and calendar sync into a single loop. Voice is captured on-device, processed in the cloud, scheduled, and mirrored back to both Google Calendar and the pager's display.

Architecture

1) Audio Capture: Raspberry Pi 4B with I2S mic plus push-to-talk button and status LED.
2) Interpret: Flask/FastAPI server transcribes with Whisper and structures events/tasks/study blocks with GPT-4.
3) Schedule/Optimize: Hard events (fixed, mirrored from Google Calendar), tasks (e.g., homework with due dates, durations, priorities), and study blocks (movable, created to satisfy tasks) are placed by a rule-based/LLM scheduler. It enforces hard events as immovable constraints so study blocks never overlap them. When users shift study blocks in Google Calendar, the scheduler recomputes placements to fill gaps and preserve deadlines.
4) Sync: Bidirectional sync keeps hard events and study blocks aligned with Google Calendar.
5) Display: E-ink pager shows "Next" and "Today's Agenda" with lift-to-wake and scrolling for the full agenda.
6) Web UI: Browser GUI to review and play uploads, see metadata, and trigger reprocessing.
7) Audio Output: Speaker playback of short, LLM-planned summaries to inform the user what just happened (e.g., confirmation, next steps).

Technical Components

  • Hardware: Raspberry Pi 4B, SPH0645 I2S mic, push-to-talk button (GPIO 11), status LED (GPIO 26), optional e-ink display.
  • Firmware: Python client (`pi/audio_capture_rpi.py`) handles GPIO, in-memory WAV encoding, and HTTP upload.
  • Backend: Flask/FastAPI server (`audioCapture_server.py`) for uploads, Whisper STT, GPT-based parsing, scheduler, and Google Calendar sync.
  • Data: Google Calendar is the user-facing source for hard events and study blocks; scheduler state tracks tasks and block metadata for optimization.

Hardware Prototype

Prototype

The current build records up to 30 seconds of audio, auto-detects microphones, provides LED feedback for recording/upload states, and posts directly to the server. The server hosts a simple web UI to view, play, and download recordings, and exposes API endpoints for agenda queries and reprocessing.

Results

SmartPager reliably captures and uploads voice notes, converts them to structured tasks/events, and schedules study blocks that respect hard calendar events. Agenda endpoints (`/api/agenda/next`, `/api/agenda/today`) serve the merged view to the pager client.

SmartPager - Beginner Demo!

This demo shows the full voice-driven loop: querying Monday’s agenda on the e-ink pager, adding a study block, deleting an existing event, handling a dentist request that conflicts with the new study block, proposing and accepting a new time, and rebooking—demonstrating capture → LLM parsing → conflict-aware scheduling → calendar update → on-device display.

SmartPager Demo - Display Options

This demo shows the pager pulling up day-by-day agendas (Mon–Sun) on the e-ink display, adding a study block (Mon 3–4pm), deleting an existing event (Mon dinner), detecting a hard conflict (dentist vs study block), suggesting a new time, and re-booking the appointment—all while navigating schedules with the on-device buttons and scrolling feature.

SmartPager Demo - Syncing with Google Calendar

This demo proves live, bidirectional sync: spoken requests create new events in Google Calendar, events can be moved (gym shifted to 5 p.m.), and cancellations propagate (signals and circuits removed), with changes appearing immediately in the user's personal Google Calendar account.

SmartPager Demo - Resolving Conflicts

This demo shows the system catching a lunch request that conflicts with a fixed dentist appointment, proposing a new time, and booking it after the user agrees while the web UI (our lightweight calendar view) reflects the resolved schedule.
